# 销售模块工作台化重构设计

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 将销售模块从"左右分栏购物车模式"重构为"全页工作台填表模式"，解决商品行多时信息密集、视线跳转、操作步骤繁琐的问题。

**Architecture:** 取消左右分栏，改为顶栏元信息 + 全宽可编辑表格 + 底栏汇总的三段式布局。商品通过表格末尾空行的内联搜索添加，搜索下拉按仓库/仓位展开可直接点选预填仓库。

**Tech Stack:** Vue 3 Composition API + Tailwind CSS 4 + 项目现有 UI 类名体系（card/input/badge/btn-primary 等）

---

## 1. 整体布局

三个区域，单栏全宽：

```
┌──────────────────────────────────────────────────────────┐
│ 销售管理                                                  │  ← 页面标题 h2
│ [订单类型▾] [客户搜索...] [员工▾] [关联订单...](退货时)      │  ← 顶栏元信息
├──────────────────────────────────────────────────────────┤
│ 工作台表格                                                │
│ # │ 商品      │ 数量(±)│ 单价 │ 仓库  │ 仓位 │ 小计 │ 操作 │
│ 1 │ ...       │[-]2[+]│  99  │[主仓▾]│[A1▾] │ 198 │ 拆/删│
│ + │ [搜索...]  │       │      │      │      │     │     │
├──────────────────────────────────────────────────────────┤
│ 底栏：[备注...]                       合计: ¥248  [提交]  │
└──────────────────────────────────────────────────────────┘
```

- 页面左上角显示 `<h2 class="text-lg font-bold mb-4">销售管理</h2>` 标题，与寄售管理等页面风格一致
- 顶栏固定可见，一行横排
- 表格占据主体，列：序号 / 商品（名称+SKU）/ 数量 / 单价 / 仓库 / 仓位 / 小计 / 操作
- 有 `finance` 权限时多显示成本价列（只读）
- 底栏：左侧备注输入，右侧合计+清空+提交

## 2. 内联搜索交互

表格末尾始终有一个空行，商品列为搜索输入框：

- 输入关键词，fuzzyMatchAny 匹配 SKU/名称/品牌
- 下拉面板分两级：
  - **商品标题行**（不可点击）：名称 · SKU · 品牌
  - **仓库/仓位行**（可点击）：仓库名 / 仓位码(带颜色badge) / 库存数 / 零售价
- 库存为 0 的行灰显仍可见
- 寄售仓行显示紫色 badge

**点击仓库行后：**
1. 空行填充为数据行（预填商品、零售价为默认单价、数量1、仓库、仓位）
2. 数量输入框自动聚焦
3. 下方生成新空行
4. 下拉关闭

**键盘：** ↑/↓ 导航（跳过标题行）、Enter 选中、Esc 关闭

## 3. 表格行编辑

**数据行结构：**
```
│ 1 │ iPhone 16 Pro │ [-] 2 [+] │ ¥99  │ [主仓▾] │ [A1▾] │ ¥198 │ 拆 ✕ │
│   │ IP16P         │           │      │ 库存:10 │       │      │      │
```

| 列 | 交互 |
|---|---|
| 序号 | 只读自动编号 |
| 商品 | 只读，名称+SKU |
| 数量 | [-]/[+] 按钮步进1；输入框可直接改；最小值1；退货模式 ≤ max_return_qty |
| 单价 | 直接编辑数字输入框 |
| 仓库 | 下拉选择，切换后清空仓位，选项显示可用库存 |
| 仓位 | 下拉选择，依赖仓库，选项显示可用库存 |
| 小计 | 只读，数量 × 单价 |
| 操作 | 拆分（复制行清空仓库仓位）+ 删除 |

**库存警告：** 数量 > 可用库存时库存数字变红

## 4. 订单类型适配

五种类型共用工作台，条件控制差异：

| | 现款 | 账期 | 寄售调拨 | 寄售结算 | 退货 |
|---|---|---|---|---|---|
| 客户 | 必选 | 必选 | 必选 | 必选 | 自动填入 |
| 搜索空行 | 正常 | 正常 | 正常 | 仅寄售仓 | 隐藏 |
| 仓库/仓位 | 可编辑 | 可编辑 | 可编辑 | 锁定 | 可编辑 |
| 数量限制 | ≤库存 | 无硬限制 | ≤库存 | ≤寄售库存 | ≤max_return_qty |
| 拆分按钮 | 显示 | 显示 | 显示 | 隐藏 | 隐藏 |
| 顶栏额外 | — | — | — | — | 关联订单搜索框 |

**退货特殊流程：**
1. 切换退货 → 顶栏出现关联订单搜索框，表格清空
2. 选定原单 → 客户自动填入 → 表格自动填入可退商品行（数量默认0）
3. 用户编辑退货数量，0的行提交时过滤掉

**寄售结算：** 搜索仅显示寄售仓库存，仓库/仓位锁定

## 5. 组件架构

```
SalesView.vue (瘦容器)
├── SalesToolbar.vue          — 顶栏元信息
├── SalesWorksheet.vue        — 工作台表格主体
│   ├── WorksheetRow.vue      — 数据行
│   └── ProductSearchRow.vue  — 末尾搜索空行+下拉面板
├── SalesFooter.vue           — 底栏汇总
└── OrderConfirmModal.vue     — 确认弹窗（保持不变）
```

**数据流：**
- SalesView 持有 saleForm + cart (composable)
- 各子组件通过 props 接收、emit 回传
- useSalesCart.js composable 保留核心逻辑，微调 addToCart 入参
- Store 层和后端 API 完全不动

**与现有代码关系：**
- 删除：ProductSelector.vue、ShoppingCart.vue、CartItem.vue
- 新建：SalesToolbar.vue、SalesWorksheet.vue、WorksheetRow.vue、ProductSearchRow.vue、SalesFooter.vue
- 修改：SalesView.vue（模板和部分逻辑）、useSalesCart.js（入参微调）
- 不改：OrderConfirmModal.vue、后端 API

## 6. 移动端适配

- 顶栏：字段换行堆叠
- 表格隐藏列：SKU、成本价、仓位（合并到仓库列）
- 操作列只保留删除，拆分通过长按触发
- 搜索下拉：全宽浮层，仓库行简化
- 底栏：合计和按钮堆叠两行
- 沿用现有 `md-hide` 响应式类

## 7. UI 风格约束

- 延续项目现有 Tailwind CSS 4 + 自定义类体系（card/input/badge/btn-primary 等）
- 行高舒适型，不过度紧凑
- 仓位 badge 延用颜色标记系统（badge-blue/red/green 等）
- 保持与库存页面、设置页面一致的视觉风格
