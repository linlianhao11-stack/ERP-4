# 销售退货修复 + 订单详情内退货设计

## 问题

1. 销售退货页面加载订单失败：`SalesView.vue` 中 `searchReturnOrders` 函数对后端返回的 `{ items, total }` 对象直接调用 `.filter()`，TypeError 导致加载失败
2. 退货入口不便：用户需跳转到销售页面、切换退货模式、搜索关联订单，流程繁琐
3. 期望在订单详情弹窗内直接发起退货

## 设计决策

### Bug 修复
- `SalesView.vue:196` 改 `data.filter(...)` 为 `(data.items || data).filter(...)`

### 订单详情内退货

**入口条件：**
- 订单类型为 CASH 或 CREDIT
- 订单未取消（`shipping_status !== 'cancelled'`）
- 至少有一个商品 `available_return_quantity > 0`

**退货表单（在详情弹窗内展开）：**
- 列出可退商品，每行：商品名、SKU、可退数量、退货数量输入框
- 退款方式：「已退款给客户」勾选框（勾选=已退款，不勾选=形成在账资金）
- 仓库仓位：**自动使用原订单仓库 + 该仓库第一个活跃仓位**，用户无需选择

**提交逻辑：**
- 调用现有 `POST /api/orders`（order_type=RETURN）
- 完全复用后端现有退货逻辑（库存回入、负金额、应收红字、退款单等）
- 零后端接口改动

**成功后：**
- 关闭退货表单、刷新订单列表
- Toast 提示成功

## 变更范围

| 层 | 文件 | 改动 |
|----|------|------|
| 前端 | SalesView.vue | 修复 `data.filter` → `data.items.filter` |
| 前端 | FinanceOrdersTab.vue | 新增退货表单 UI + 提交逻辑 + 退货按钮 |
| 前端 | api/orders.js | 确认 createOrder 可复用（已有） |
| 后端 | 无改动 | 完全复用现有 POST /api/orders RETURN 逻辑 |
