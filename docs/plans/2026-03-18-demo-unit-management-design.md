# 样机管理模块 — 设计方案

> 日期：2026-03-18
> 状态：待团队讨论

---

## 一、为什么要做这个模块？

公司有一批电子设备/数码产品样机（50 台以内），日常会借给客户试用、业务员带出去拜访、参加展会等。目前靠 Excel 手工记录，容易漏记、找不到谁借了哪台、超期也没人提醒。

**这个模块要解决的核心问题：**
- 每台样机在哪、谁在用、什么时候该还，一目了然
- 借出要审批，归还要检查，流程规范
- 样机不用了可以翻新卖掉、转成正常库存、或者报废，账目自动处理

---

## 二、放在哪里？

**放在「库存」页面下面，作为第二个标签页。** 打开库存页面后，顶部会有两个标签：「库存」和「样机管理」。点「样机管理」就进入样机模块。

这样做的好处：样机本来就是仓库在管的东西，不用多一个菜单入口，管仓库的人顺手就能管样机。

---

## 三、样机的一生（生命周期）

一台样机从入库到退出，会经历这些阶段：

```
买来入库 → 放在样机仓（可以借出）
                ↓
            借给别人用
                ↓
          到期了，催他还回来
                ↓
        还回来了，检查一下状态
                ↓
         ┌──────┼──────┬──────────┐
         ↓      ↓      ↓          ↓
       继续放着  翻新后   折价卖给    不能用了
       下次再借  转成正常  客户       报废处理
                库存
```

**如果样机弄丢了：** 登记丢失，向借用人收取赔偿。

## 四、核心功能

### 4.1 样机台账 — "我有哪些样机"

就像一个花名册，每台样机都有一条记录：

- **样机编号**：系统自动生成，比如 DM-20260318-001
- **对应商品**：是哪个产品（从商品库里选）
- **SN 码**：序列号，用来唯一识别这台机器（复用现有的 SN 码管理）
- **所在仓库**：属于哪个样机仓
- **当前状态**：在库 / 借出中 / 维修中 / 已卖出 / 已报废 / 已丢失 / 已转良品
- **成色**：全新 / 良好 / 一般 / 较差
- **成本价**：这台样机花了多少钱
- **当前谁在用**：如果借出了，显示借给了哪个客户或哪个员工
- **累计借出次数和天数**：这台机器历史上借了几次、总共借了多少天

页面顶部有四个统计数字：**样机总数 | 在库可借 | 借出中 | 超期未还**

### 4.2 借还记录 — "谁借了什么、什么时候还"

每次借出都会产生一条借还记录：

- **借出单号**：系统自动生成
- **哪台样机**：选择要借出的样机
- **借用类型**：客户试用 / 业务员携带 / 展会活动
- **借给谁**：选择客户或员工
- **经办人**：哪个业务员经手的
- **预计归还日期**：约定什么时候还
- **借出时的状态**：出库时机器是什么成色
- **归还时的状态**：还回来时机器是什么成色
- **归还备注**：有没有损坏、缺配件等情况

**超期的记录会高亮显示**，方便一眼看到谁还没还。

### 4.3 样机入库 — "怎么把样机录入系统"

点击「新增样机」按钮后，弹窗里先选来源，有两种方式：

**方式一：从库存转入**（仓库里已有的商品，拿一台出来当样机）

```
第一步：选择来源仓库
第二步：选择商品
第三步：填写 SN 码
        → 如果来源仓库有 SN 码管理：输入时自动模糊搜索匹配该仓库已有的 SN 码
        → 如果来源仓库没有 SN 码管理：手动输入 SN 码，系统为这台样机新建 SN 记录
第四步：选择目标样机仓、确认成色
第五步：确认入库
```

系统自动处理：来源仓库扣库存 → 样机仓加库存 → 成本价自动取原仓库的加权成本 → 创建样机台账记录

**方式二：新采购入库**（新买的，直接入样机仓）

```
第一步：选择商品
第二步：手动输入 SN 码（系统新建 SN 记录）
第三步：填写成本价
第四步：选择目标样机仓、确认成色
第五步：确认入库
```

系统自动处理：样机仓创建库存 → 创建样机台账记录

**关于 SN 码的说明：** 不管样机从哪里来，最终每台样机都会有 SN 码用于追踪。区别只是：如果原仓库本身就管 SN 码，输入时可以搜索匹配已有的；如果原仓库不管 SN 码，就手动输入一个新的。

### 4.4 借出流程

```
第一步：业务员创建借出申请
        ↓ 填写借哪台、借给谁、预计什么时候还、借出目的
第二步：主管审批
        ↓ 同意或拒绝
第三步：仓管确认出库
        ↓ 记录借出日期和出库时的机器状态
第四步：等待归还
        ↓ 系统跟踪，超期自动提醒
```

### 4.5 归还流程

```
第一步：借用人把样机拿回来
第二步：仓管检查机器状态
        ↓ 选择归还成色（全新/良好/一般/较差）
        ↓ 如有损坏，写在备注里
第三步：确认归还，样机回到「在库」状态
```

### 4.6 四种退出路径 — "样机不当样机了"

#### 路径一：转销售（折价卖给客户）

客户试用后想买下来，或者样机用旧了折价出售。

操作：填写售价和买家 → 系统自动生成一笔销售订单 → 自动生成应收账款 → 后续走正常收款流程

#### 路径二：翻新转良品（翻新后放回正常库存卖）

样机翻新整备后，转成正常商品库存。

操作：选择转入哪个仓库/仓位 → 填写翻新费用（可选） → 确认后系统自动把这台机器从样机仓移到正常仓库 → 入库成本 = 样机原价 + 翻新费用 → SN 码跟着一起转过去

#### 路径三：报废

样机老化或损坏严重，没有维修价值了。

操作：填写报废原因和残值 → 系统自动做账（从库存里核销掉） → 样机标记为已报废

#### 路径四：丢失赔偿

样机弄丢了，需要向借用人追偿。

操作：填写丢失说明和赔偿金额 → 系统自动生成一笔应收账款（向借用人收钱） → 自动做账 → 样机标记为已丢失

## 五、一些重要规则

1. **一台机器同时只能借给一个人** — 不能重复借出
2. **借出中的样机只能做"转销售"或"登记丢失"** — 不能直接报废或翻新
3. **只有在库的样机才能翻新转良品** — 得先还回来
4. **到了终态就不能再操作了** — 卖掉了/报废了/丢了/转良品了，这台样机的生命周期就结束了
5. **权限复用库存模块** — 能看库存的人就能看样机，能编辑库存的人就能操作样机

## 六、需要记录哪些数据

### 样机信息（每台一条）

| 记什么 | 说明 |
|--------|------|
| 样机编号 | 系统自动编号 |
| 对应商品 | 是哪个产品 |
| SN 码 | 序列号 |
| 所属样机仓 | 放在哪个仓库 |
| 当前状态 | 在库/借出/维修中/已卖出/已报废/已丢失/已转良品 |
| 成色 | 全新/良好/一般/较差 |
| 成本价 | 当初花了多少钱 |
| 当前持有人 | 谁在用（客户名或员工名） |
| 累计借出次数 | 历史借了几次 |
| 累计借出天数 | 历史借了多少天 |
| 备注 | 补充说明 |

### 借还记录（每次借出一条）

| 记什么 | 说明 |
|--------|------|
| 借出单号 | 系统自动编号 |
| 哪台样机 | 关联样机编号 |
| 借用类型 | 客户试用/业务员携带/展会 |
| 借给谁 | 客户名或员工名 |
| 经办业务员 | 谁经手的 |
| 审批人 | 谁批的 |
| 借出日期 | 什么时候拿走的 |
| 预计归还日期 | 约定什么时候还 |
| 实际归还日期 | 实际什么时候还的 |
| 借出时成色 | 拿走时机器什么状态 |
| 归还时成色 | 还回来时什么状态 |
| 归还备注 | 有没有损坏等 |
| 借出目的 | 为什么借 |

### 处置记录（退出时一条）

| 记什么 | 说明 |
|--------|------|
| 哪台样机 | 关联样机编号 |
| 处置方式 | 转销售/报废/丢失赔偿/翻新转良品 |
| 金额 | 售价/残值/赔偿金额 |
| 翻新费用 | 翻新转良品时的翻新花费 |
| 转入仓库 | 翻新转良品时放到哪个仓库 |
| 关联订单 | 转销售时关联哪笔销售订单 |
| 处置原因 | 为什么这样处理 |
| 审批人 | 谁批的 |

## 七、和其他模块怎么配合

| 配合谁 | 怎么配合 |
|--------|----------|
| 库存管理 | 样机入库出库都走库存系统，成本正常计算 |
| SN 码管理 | 样机绑 SN 码，复用现有体系 |
| 客户管理 | 借给客户时从客户列表里选 |
| 员工管理 | 经办人、借用员工从员工列表里选 |
| 销售订单 | 转销售时自动生成销售订单 |
| 应收账款 | 转销售和丢失赔偿时自动生成应收单 |
| 会计凭证 | 报废和丢失时自动生成记账凭证 |

## 九、开发范围估算

**新增文件：** 约 12 个（后端 4 个 + 前端 7 个 + 数据库 1 个）

**修改现有文件：** 3 个（库存页面加标签、注册新模块）

**不影响现有功能：** 库存、销售、财务等模块的现有功能完全不变，只是在库存页面多了一个标签页。
