# 阶段3：应收应付管理（AR/AP）— 完成记录

> 状态：✅ 已完成（2026-02-28）
> 分支：feature/phase3-ar-ap → 已合并到 main

## 概述

建立完整的应收/应付体系（7个新模型），与现有业务流程（发货/收货/退货/付款）自动衔接，实现财务层应收应付的独立管理。

## 关键设计决策

| 决策 | 选择 | 说明 |
|------|------|------|
| ReceiptBill vs Payment | 并存+关联 | Payment 保留业务层（Customer.balance），ReceiptBill 记财务层（账套隔离），通过 payment_id FK 关联 |
| 应收单生成时机 | 全部发完 + 支持手动 | shipping_status="completed" 时自动生成；另提供手动创建入口 |
| 权限 | 新增6个 | accounting_ar_view/edit/confirm + accounting_ap_view/edit/confirm |
| 单据编号 | generate_order_no(前缀) | YS=应收/SK=收款/SKTK=收款退款/YSHX=核销/YF=应付/FK=付款/FKTK=付款退款 |
| 凭证关联 | 期末批量生成 | 确认后的收款/付款/退款/核销单在期末批量生成凭证 |

## 业务流程触发规则

| 触发场景 | 钩子位置 | 自动生成 |
|----------|---------|---------|
| CASH 订单全部发货 | logistics.py ship_order_items | ReceivableBill(completed) + ReceiptBill(confirmed, 关联Payment) |
| CREDIT 订单全部发货 | logistics.py ship_order_items | ReceivableBill(pending) |
| CONSIGN_SETTLE 全部发货 | logistics.py ship_order_items | ReceivableBill(pending) |
| RETURN 退货单创建 | orders.py create_order | ReceivableBill(completed, 负数红字) |
| 采购订单收货 | purchase_orders.py receive | PayableBill(pending) |
| 采购订单付款 | purchase_orders.py pay | DisbursementBill(confirmed) |

## 7个模型

1. **ReceivableBill（应收单）** — 记录应收金额，关联 Order，tracked received/unreceived
2. **ReceiptBill（收款单）** — 关联 Payment + ReceivableBill，draft→confirmed
3. **ReceiptRefundBill（收款退款单）** — 关联原收款单，确认时回滚应收
4. **ReceivableWriteOff（应收核销单）** — 预收冲应收，关联 advance_receipt + receivable_bill
5. **PayableBill（应付单）** — 记录应付金额，关联 PurchaseOrder，tracked paid/unpaid
6. **DisbursementBill（付款单）** — 关联 PayableBill，draft→confirmed
7. **DisbursementRefundBill（付款退款单）** — 关联原付款单，确认时回滚应付
